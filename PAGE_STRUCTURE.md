# ページ構造とナビゲーションフロー

## 現在のメニュー構成

### MAIN MENU

1. **プロジェクト** (`/project`)

   - 今日の予定カード（新規追加）

5. **UTM**（サブメニュー）

## ページ関係図

### パターンA: 既存UTMフロー（従来のワークフロー）

```
UTM > UTMダッシュボード
  ├─ 飛行前準備
  ├─ 飛行エリア
  └─ ワークフロー
```

### パターンB: 新規プロジェクトベースフロー（PR #148で追加）

```
プロジェクト一覧
  │
  ├─ 今日の予定カード
  │   ├─ 「開始」ボタン → プロジェクト詳細（Preflightタブ）
  │   ├─ 「監視」ボタン → 運航監視
  │   └─ 「レポート」ボタン → ポストフライト
  │
  └─ プロジェクトカードクリック
      ↓
  プロジェクト詳細 (/project/:id)
      ├─ Preflightタブ（新規）
      │   ├─ プリフライトチェック
      │   └─ フライト開始ボタン → 運航監視
      ├─ 概要タブ
      ├─ エリアタブ
      ├─ デバイスタブ
      ├─ メンバータブ
      ├─ 飛行ログタブ
      └─ メモタブ
      ↓
  運航監視 (/monitoring/:flightId)
      ├─ ドローンリスト
      ├─ マップビュー
      ├─ テレメトリー
      └─ フライト完了ボタン → ポストフライト
      ↓
  ポストフライト (/postflight/:flightId)
      ├─ フライトサマリー
      ├─ インシデント記録
      ├─ ノート
      ├─ 添付ファイル
      └─ 送信 → プロジェクト一覧
```

## 問題点と改善案

### 現在の問題

1. **プロジェクト詳細ページの構成**
   - 現状: プロジェクト詳細ページ (`/project/:id`) がフルスクリーンでマップを表示
   - 問題: プロジェクト一覧からクリックすると、いきなり大きなマップが表示され、何のページか分かりにくい

2. **ページの役割が不明確**
   - プロジェクト詳細ページが「プロジェクト管理」なのか「フライト準備」なのか曖昧
   - Preflightタブとエリアタブの違いが不明確

3. **既存UTMと新規フローの関係性**
   - UTMダッシュボードと新規の運航監視ページの使い分けが不明
   - 既存UTMの「飛行前準備」と新規の「Preflightタブ」の違いが不明

---

## 改善提案

### 提案1: プロジェクト詳細ページのデフォルトタブ変更

**現在**:

- デフォルトでマップが表示される（エリアタブ相当？）

**改善案**:

- デフォルトで「概要」タブを表示
- マップは「エリア」タブに配置
- URLパラメータがない場合は `/project/:id?tab=overview` にリダイレクト

### 提案2: タブ構成の見直し

**現在のタブ順**:

1. Preflight（新規）
2. 概要
3. エリア（マップ）
4. デバイス
5. メンバー
6. 飛行ログ
7. メモ

**改善案A - プロジェクト管理重視**:

1. 概要（デフォルト）
2. エリア（マップ）
3. デバイス
4. メンバー
5. Preflight
6. 飛行ログ
7. メモ

**改善案B - フライト運用重視**:

1. 概要（デフォルト）
2. Preflight
3. エリア（マップ）
4. 飛行ログ
5. デバイス
6. メンバー
7. メモ

### 提案3: ページ分離

**改善案**:

- プロジェクト詳細ページ: プロジェクト管理に特化
  - 概要、エリア、デバイス、メンバー、メモ
- フライト準備ページ（新規）: フライト運用に特化
  - Preflight、運航監視、ポストフライト、飛行ログ

**ナビゲーション**:

```
プロジェクト詳細
  ├─ 「フライト準備を開始」ボタン
  └─ → フライト準備ページ (/flight-prep/:projectId)
      └─ Preflightタブ
          └─ 「フライト開始」ボタン
              └─ → 運航監視 (/monitoring/:flightId)
```

### 提案4: 既存UTMとの統合

**改善案**:

- UTM > 飛行前準備 → 新規Preflightタブにリダイレクト
- UTM > 飛行エリア → プロジェクト一覧にリダイレクト
- UTM > ワークフロー → プロジェクト詳細にリダイレクト
- UTMダッシュボード → そのまま維持（または新規運航監視にマージ）

## 推奨フロー

### オプションA: シンプル統合（推奨）

```
プロジェクト一覧
  ↓ プロジェクト選択
プロジェクト詳細（概要タブがデフォルト）
  ↓ Preflightタブに切り替え
プリフライトチェック
  ↓ フライト開始ボタン
運航監視
  ↓ フライト完了ボタン
ポストフライト
  ↓ 送信
プロジェクト一覧
```

**変更点**:

- プロジェクト詳細のデフォルトタブを「概要」に変更
- マップは「エリア」タブに配置
- URLパラメータなしの場合、`?tab=overview` を追加

### オプションB: 2段階フロー

```
プロジェクト一覧
  ↓ プロジェクト選択
プロジェクト詳細（管理画面）
  ↓ 「フライト準備」ボタン
フライト準備ページ（新規）
  ↓ プリフライトチェック
  ↓ フライト開始ボタン
運航監視
  ↓ フライト完了ボタン
ポストフライト
  ↓ 送信
プロジェクト一覧
```

**変更点**:

- フライト準備ページを新規作成（`/flight-prep/:projectId`）
- Preflightタブをフライト準備ページに移動
- プロジェクト詳細は純粋な管理画面に
