## ルールの適用

これらのルールは以下の方法で適用されます：

1. **ESLint**: 自動的なコード品質チェック
2. **Prettier**: コードフォーマットの統一
3. **TypeScript**: 型安全性の確保
4. **Storybook**: コンポーネントドキュメントの管理

## ルールの更新

新しいルールや変更が必要な場合は、以下の手順で更新してください：

1. 該当するルールファイルを編集
2. 必要に応じてESLint設定を更新
3. チームメンバーに変更を通知
4. Storybookのドキュメントも同時に更新
